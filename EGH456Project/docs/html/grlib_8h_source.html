292// For the benefit of those of you reading the source...
293
294// Note that, unlike tFont and tFontEx where the character data and offset
295// tables are referenced by pointer and may be discontiguous, a font described
296// using tFontWide is assumed to comprise a single, contiguous block of data
297// with a tFontWide structure as its header, a number of tFontBlock structures
298// immediately following this, and a number of tFontOffsetTable and font glyph
299// data entries following this. This format ensures that the font is position-
300// independent and allows use in external memory or from non-random-access
301// storage such as SDCards or SSI flash.
302//
303// A complete font in this format would look as follows:
304//
305// -------------------------------
306// 0 | tFontWide header |
307// | ui32NumBlocks = n |
308// -------------------------------
309// | tFontBlock[0] | First block describes characters from
310// | ui32StartCodepoint = S0 | codepoint S0 to (S0 + N0 - 1). Offset
311// | ui32NumCodepoints = N0 | to glyph table is OF0 bytes from the
312// | ui32GlyphTableOffset = OF0 | start of the tFontWide header struct.
313// -------------------------------
314// | ... |
315// -------------------------------
316// | tFontBlock[n-1] | Final block describes characters from
317// | ui32StartCodepoint = S(n-1) | codepoint S(n-1) to (S(n-1) + N(n-1)
318// | ui32NumCodepoints = N(n-1) | - 1).
319// | ui32GlyphTableOffset = OF(n-1)|
320// -------------------------------
321// Start of Block 0 Offset Table
322// -------------------------------
323// OF0 | ui32GlyphOffset = GO0 | Offset from OF0 to glyph S0 data.
324// | |
325// -------------------------------
326// | ui32GlyphOffset = GO1 | Offset from OF0 to glyph (S0 + 1) data.
327// | |
328// -------------------------------
329// | ... |
330// | |
331// -------------------------------
332// | ui32GlyphOffset = GO(N0 - 1) | Offset from OF0 to glyph (S0 + N0 - 1)
333// | |
334// -------------------------------
335// Start of Block 0 Glyph Data
336// -------------------------------
337// GO0 | Glyph data for codepoint |
338// | S0 from block 0 |
339// -------------------------------
340// GO1 | Glyph data for codepoint |
341// | S0 + 1 from block 0 |
342// -------------------------------
343// | ... |
344// -------------------------------
345// GO | Glyph data for codepoint |
346//(N-1)| (S0 + N0 - 1) from block 0 |
347// -------------------------------
348// Start of Block 1 Offset Table
349// -------------------------------
350// OF1 | ui32GlyphOffset = G10 | Offset from OF1 to glyph S1 data.
351// | |
352// -------------------------------
353// | ui32GlyphOffset = G11 | Offset from OF1 to glyph (S1 + 1) data.
354// | |
355// -------------------------------
356// | ... |
357// | |
358// -------------------------------
359// | ui32GlyphOffset = G1(N1 - 1) | Offset from OF1 to glyph (S1 + N1 - 1)
360// | |
361// -------------------------------
362// Start of Block 1 Glyph Data
363// -------------------------------
364// G10 | Glyph data for codepoint |
365// | S1 from block 1 |
366// -------------------------------
367// G11 | Glyph data for codepoint |
368// | S1 + 1 from block 1 |
369// -------------------------------
370// | ... |
371// -------------------------------
372// G1 | Glyph data for codepoint |
373//(N-1)| (S1 + N1 - 1) from block 1 |
374// -------------------------------
375// GO0 | Glyph data for codepoint |
376// | S0 from block 0 |
377// -------------------------------
378// Start of Block 2 Offset Table
379// -------------------------------
380// | ...continued for remaining |
381// | font blocks. |
382// -------------------------------
383//
